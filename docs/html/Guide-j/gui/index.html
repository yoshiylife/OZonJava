グラフィカルユーザインタフェースのプログラミング


一般的に、グラフィカルユーザインタフェース（GUI）のプログラミングはその構成要素が多く複雑なものになります。その結果学習するのが困難になり、新しいGUIインタフェースの採用は現状利用しているGUIとのトレードオフになります。また、GUIがアプリケーションに与える影響も多大です。
OZシステムではイベント配送などについて独自の方法を用いていますが、プログラマの便宜のために、Java1.1のAWT（Abstract Window Toolkit）と同一のインタフェースを提供しています。したがって、JavaのAWTを利用してプログラミングをされたかたには、GUIを持つOZアプリケーションの構築は容易なものでしょう。また、Javaのプログラミングテキストは多数出版されているので、それを利用してGUIを持つアプリケーション構築のための知識を得ることができるでしょう。


ここでは、GUIを持つOZアプリケーションを構築する上でのOZ独特の要素をご紹介しています。

GUIプログラミングのためのクラスはOZ標準クラスライブラリの「GUIライブラリ」に含まれています。
