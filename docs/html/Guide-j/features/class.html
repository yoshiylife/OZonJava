OZシステムにおけるクラス


OZ言語のクラス

OZ言語の中のクラスは、2つのタイプで宣言されます。

  • class
  • cell

どちらもオブジェクト指向プログラミングでのクラスを表現します。

この2つのクラスはOzSecureクラスを継承します。OzSecureクラスはすべてのOZクラスの親クラスを継承します。
class型クラスは、OzSecureクラスを継承するRootクラスを継承するクラスです。
それに対してcell型クラスは、OzSecureクラスを継承するOzCellクラスを継承するクラスです。


クラスとOｚVMで生成されるオブジェクトの関係

OzVMはOZシステムの仮想マシンです。OzVMがクラスからオブジェクトを生成し活性化します。

cell型クラスはグローバルオブジェクトを生成します。それに対して、class型クラスはローカルオブジェクトを生成します。グローバルオブジェクトはネットワークを介してコミュニケーションする能力を持ったオブジェクトです。ローカルオブジェクトは、ローカル環境で活動するオブジェクトです。

多くのオブジェクト指向プログラミング言語では、フラットなオブジェクトモデルを採用し、すべてのオブジェクトを同一手段によりアクセスしています。 しかし、この方法で分散環境で動作するプログラムを作成すると、実行性能が低下する可能性があります。ネットワークを介するオブジェクトアクセスがその原因です。
OZでは、この状況を回避するために、ローカルアクセスとネットワークアクセスを区別する計算モデルを採用しました。そして、 パフォーマンス的にコストの高いネットワーク経由のアクセスは必要が発生した場合だけに行います。
ネットワークを経由するオブジェクトへのアクセスと ローカルなオブジェクトへのアクセスを静的に区別するために、 グローバルオブジェクトとローカルオブジェクトというの2種類のオブジェクトを導入し、これをOZ言語の型システムに組み込んでいます。


OｚＶＭのクラス管理

OzVMではクラスを識別子で管理しています。
OZ言語のクラスの名前はコンパイル単位でユニークであることが要求されます。 別のコンパイル単位では、同一の名前のクラスが存在していてもかまいません。 しかし、分散環境であるOZでは、1つのクラスの定義が世界中でユニークなものでなければなりません。そこで、クラス名とは別にクラス識別子を付与し、それによってクラスのアイデンティティを確立します。
したがって、ネットワークでつながった世界規模のOZシステムの中で同じ名前を持つクラスを複数定義できます。また、インタフェース定義とクラス本体の定義を別々に管理することができます。
