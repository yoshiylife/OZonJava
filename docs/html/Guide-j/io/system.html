１．標準出力
標準出力に出力するためには、Systemクラスのオブジェクトを生成し、それに対してprintlnメッセージを送ります（＝Systemuクラスのオブジェクトのprintlnメソッドを起動する）。

Systemクラスのprintlnメソッド 
        public void printMsg() {
            System system;
            system->println("Hello1");
        }

ここでは文字列を引数にしてその文字列を出力をしていますが、asStringメソッドが定義されているオブジェクトであればどのようなオブジェクトでも引数にすることができます。
String型のオブジェクトが引数に指定された場合にはその文字列が標準出力に指定され、その他の場合は引数オブジェクトのasStringメソッドで獲得される文字列表現が出力されます。



２．Systemクラス

上位クラス： *Root

コンストラクタメンバ
new create() コンストラクタ。

主なメソッドメンバ
public final void println(Root arg1) オブジェクトarg1のasString()メソッドで得られる文字列を
標準出力に出力します。
public final void println(String arg1) 文字列arg1を標準出力に出力します。
public final void debugPrintln(Root arg1) デバッグモードで、オブジェクトarg1のasString()メソッ
ドで得られる文字列を標準出力に出力します。
public final void debugPrintln(String arg1) デバッグモードで、文字列arg1を標準出力に出力する。
public final InputStreamReader getStdin() 標準テキスト入力リーダーを答えます。
public final PrintWriter getStdout() 標準テキスト出力ライターを答えます。
public final PrintWriter getStderr() 標準エラーテキスト出力ライターを答えます。
public final String getOzHome() OZホームを表現する文字列を答えます。

標準出力にメッセージを出力するだけでなく、Systemクラスのオブジェクトは標準入力、標準エラー出力を保持し、その問い合わせに答えることができます。

（APIの詳細は「ツールリファレンス」の「クラスライブラリ」の該当箇所を参照してください。）



３．キーボードからの入力（標準入力）

キーボードからの入力（標準入力）を受け付けるためには、まず、Systemクラスのオブジェクトに標準入力を問い合わせます。getStdinメソッドがそのサービスを行います。

標準入力はInputStreamReaderクラスのオブジェクトです。そのオブジェクトが持つreadメソッドを起動することで標準入力からの入力を受け付けることができます。

SystemクラスのgetStdinメソッド 
    System sys;
    CharArray sa=>create(1);
    int i = 0;

    while(true){
        if((i=sys->getStdin()->read(sa,0,1))==0){
        s->debugPrintln("read fault");break;
    }

（APIの詳細は「ツールリファレンス」の「クラスライブラリ」の該当箇所を参照してください。）



４．Systemオブジェクトを使用するデバッグ

OzVMにはグローバルオブジェクトをデバックモードで稼動させる機能があります。debugPrintlnメソッドを利用するとデバックモードで稼動しているときのみメッセージ出力を行うことができます。

SystemクラスのdebugPrintlnメソッド 
    ...
    OutputStream os = socket->getOutputStream ();
    OutputStreamWriter ow=>breed (os);

    String str;
    /* server was abort */
    if (html == null) {
      HTMLData html=>create ();
      html->startOutput ("text/html");
      html->beginTag ("html");
      html->addOutput ("OZCGI server stopped.");
      html->endOutput ();
      str = html->output ();
    } else
      str = html->output ();

    System sys=>create ();
    sys->debugPrintln ("sending data: ");
    sys->debugPrintln (str);

    ow->write (str, 0, str->length ());
    ow->flush ();
    ...

グローバルオブジェクト単位でデバックモードの指定ができます。

まず、OzVMをインタラクティブモードで起動します。

インタラクティブモード 
    > oz -i
    Logging file F:\oz\logs\dorothy01090.log   
    ...
    Executor>

そして、デバックしたいグローバルオブジェクトを活性化して、そのグローバルオブジェクトををデバックモードにします。グローバルオブジェクトの指定はGODで行います。

デバックモード 
    Executor>go <GOD>
    Executor>debug <GOD> TRUE
    Executor>

SystemクラスのオブジェクトへdebugPrintlnメッセージが送信される（=debugPrintlnメソッドが起動される）と、標準出力にメッセージが表示されます。

例 
    > oz -i
    Logging file F:\oz\logs\dorothy01090.log   
    ...
    Executor>go htmltest
    Executor>debug htmltest TRUE
    Executor>sending data:
    text/html 
    ... 
    OZCGI server stopped.
(リターンキーで、Executor> プロンプトが表示されます)
