名前とスコープ


ここでは、名前のスコープについて説明しています。


OZ 言語では、以下の識別子はすべて異なる名前空間を持ちます。

  • クラス名
  • メソッド名およびコンストラクタ名
  • アトリビュート名
  • メソッド、コンストラクタ、catch 節の仮引数名およびローカル変数名
  • ラベル



1. クラス名

コンパイル単位がクラス名のスコープです。同一コ ンパイル単位では、複数のクラスに同一クラス名をつけることはできません。


2. メソッド名とコンストラクタ名

メソッドとコンストラクタの名前のスコープは、それが定義されているクラスです。
メソッドは上位クラスの定義を再定義することができます。 コンストラクタは上位クラスの定義を隠蔽することができます。
また、メソッドとコンストラクタは多重定義できます。すなわち、同一名称でシグネチャの異なるメソッドの本体を定義することができます。
メソッド名とコンストラクタ名は同一名前空間を占有するので、メソッド名に等しいコンストラクタ名を定義することはできません。


3. アトリビュート名

定義されているクラスがアトリビュート名のスコープになります。
アトリビュートは上位クラスの定義を隠蔽することができます。


4. メソッドおよびコンストラクタの仮引数名

仮引数名が定義されているメソッドあるいはコンストラクタ本体の定義の終わりまでがそのスコープです。


5. catch 節の仮引数名

この仮引数名は、それが定義されている catch 節を含むブロックがスコープになります。
したがって、catch 節と同じブロックの宣言文で宣言されたローカル変数と同じ名前の仮引数を宣言することはできません。


6. ローカル変数名

宣言された位置からそのブロックの終わりまでが、そのローカル変数の名前のスコープです。


7. for 文の初期化文で宣言されるローカル変数名

このローカル変数の名前は、for 文に続 く文の終わりまでがスコープです。

したがって、以下の定義は不正です。
      for (int i = 0; i < 10; i++);
      int j = i; /* エラー! スコープ外 */


      for (int i = 0; i < 10; i++) {
          int j = i; /* OK */
      }
      int j = i; /* エラー! スコープ外 */

また catch 節の仮引数名と同様、そのスコープはその for 文含むスコープの一部となります。つまり、for 文と同じブロックの宣言文で宣言されたローカル変数と 同じ名前の仮引数を宣言することはできません。


8. ラベル

そのラベル文の終わりまでがラベルのスコープです。
すなわち、ラベルを指定した文の翻訳が終了すると、その名称空間から削除されます。
