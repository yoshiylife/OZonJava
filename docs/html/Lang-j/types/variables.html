変数（Variables）


変数とはプログラムにおけるストレージ（storage）で、必ず1つの型を持っています。


型に注目すると、変数は2種類に分類することができます。基本型の変数とクラス型の変数です。基本型（primitive type） の変数は、１つの基本型の値を常時保持しています。クラス型の変数は、変数の型に適合するクラス型のオブジェクトへのリファレンス、あるいは、nullを保持しています。


使用方法に注目すると、変数を5種類に分類することができます。

  • インスタンス変数
    インスタンス変数は、クラス宣言の中のアトリビュートのことです。

  • メソッド引数
    メソッドに引き渡される値の名前として定義されます。

  • コンストラクタ引数
    コンストラクタに引き渡される値の名前として定義されます。

  • 例外ハンドラ引数
    try文のcatch節で宣言されます。

  • ローカル変数
    ローカル変数宣言文あるいはfor文の中で宣言された変数です。




変数の初期化

インスタンス変数はオブジェクト生成時に初期化されます。

基本型とクラス型のデフォルトの初期値を以下の表に示します。

型 デフォルト値
byte型 0
short型 0
int型 0
long型 0
float型 0.0f
double型 0.0
char型 \u0000
bool型 false
クラス型 null




その他の変数の初期化を説明します。

インスタンス変数
そのクラスのメソッド本体の定義、コンストラクタ本体の定義中で有効です。

メソッド引数
メソッドが起動された（invoke）ときに、引き渡された値で初期化されます。

コンストラクタ引数
インスタンス生成式、あるいは、明示的なコンストラクタの呼出しで初期化されます。

例外ハンドラ引数
例外を表わすオブジェクトで初期化されます。

ローカル変数
宣言時の初期化（initialization）、あるいは、代入（assignment）によって値が設定されます。




変数の名前のスコープ
変数の名前は種類ごとに異なるスコープを持っています。

メソッド引数名
そのメソッド本体の定義の中で有効です。

コンストラクタ引数名
そのコンストラクタ本体の定義の中で有効です。

例外ハンドラ引数名
そのcatch節の複文で有効です。

ローカル変数
宣言された複文で有効です。（但し、その宣言文以降に有効になります。）




変数の動的な型

クラス型は動的な型を持つと言われます。クラス型は実行時に宣言クラス型のインスタンスだけではなくその下位クラスのインスタンスも参照することができます。
それに対して基本型の変数は、実行時にも宣言されたときと同じ型を持っています。
