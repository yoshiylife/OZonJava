クラス型（Class Types）


クラス型とはオブジェクトを参照とする型のことです。OZシステムでは、あらゆるオブジェクトがクラス型のインスタンスとして動的に生成されます。


OZ言語におけるクラス型は

  • class 型 (class type)
  • cell 型 (cell type)

に分類できます。cell 型のインスタンスはネットワークを介してメッセージ送受信を行う振舞いを持つことができます。それに対して、class 型のインスタンスは、ローカル環境でのみ活動できるオブジェクトです。ソースプログラムにおけるclass型とcell型の定義の違いは、クラス宣言とコンストラクタ宣言の記述可能性に限定されます。

また、クラス型を

  • システム定義のクラス型
  • 利用者定義のクラス型

と区別することができます。システム定義のクラス型は、クラスライブラリとして提供されています。




型の値の範囲

クラス型は以下の値を持つことができます

  • クラス型への参照
  • null



型がもつ操作

すべてのクラス型は以下の操作を持っています。

  • アトリビュートアクセス（attribute access）
  • メソッド起動（method invocation）
  • 等値演算子（reference equality operators）： ==、 !=
    但し、リファレンスの等値性を調べる演算が行われます。
  • 条件演算子（condition operator ） ： ? :



参考）:
クラス型の代入はすべてリファレンスのコピーです。
但し、cell型のメソッド起動（method invocation）はその規定にあてはまりません。この場合には、クラス型識別子の実引数代入に値のコピーが行なわれます。すなわち、実引数のオブジェクトのコピーが作成され、そのコピーのリファレンスが実引数として引き渡されるという仕組みになっています。

クラス型の詳細については「クラス」を参照してください。



例: クラス型の宣言 

 class Authenticator {
    ....
    public void authorize(Root obj)
    {
        turnGreen(obj);
        obj->toGreen();
    }
}
  • キーワード class を使って宣言します。




例: セル型の宣言 

 cell FedRoot {
    protected System system;
    protected Authenticator auth;
    protected boolean debug;
    ....
}
  • キーワード cell を使って宣言します。
