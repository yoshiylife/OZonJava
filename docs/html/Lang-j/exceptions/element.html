例外処理のための言語要素



例外

OZシステムにおける例外はオブジェクトとして表現されます。任意のオブジェクトを例外として扱うことができます。
システム例外をインスタンスとするクラスは、OZクラスライブラリとして提供されています。


throw文

throw文を用いると、プログラム内の任意の場所で例外を発生させることができます。ｔhrow文は任意のオブジェクトを１つ投げる(throw)ことができ、それが例外として扱われます。


例外の送出 
    UserException  e;
    
    if (exp1 == null) {
      e => create("exp1 failed.");
      throw e;
    }
UserExceptionというクラスのオブジェクトを生成し、exp1の値がnullの場合に例外を発生させています。このUserExceptionクラスは以下のように定義されています。

例外のユーザ定義 
    class UserException: *RuntimeException {
        new create(String msg) {
            super->init(msg);
        }
    }


try 文

try文により任意の例外を捕捉することができ、補足した例外に対して処理を施すことができます。


try文は以下のように記述します。

例外処理のプログラミング 
  try {
      文1;
      ...
      文n;
    } catch (例外A e) {
      // 例外Aが発生した場合の処理
      文A1;
      ...
      文A2;
    } catch (例外B e) { 
      // 例外Bが発生した場合の処理
      文B1;
       ... 
      文B2;
    } catch (NullPointerException e) {
      // システム例外
      文S1;
       ... 
      文S2;
    }
    finally {
      // 例外が発生した場合に必ず実行される処理
      文F1;
       ...
      文F2;
    }
  }


  1. 文1から文nに向かって順番に処理が行われます。途中で例外が発生したら、それが捕獲されます。
  2. 捕獲した例外の型がAであると、文A1; ... 文A2;、そして、文F1; ... 文F2;が実行されます。
  3. 捕獲した例外の型がBであると、文B1; ... 文B2;、そして、文F1; ... 文F2;が実行されます。
  4. 捕獲した例外の型がNullPointerExceptionであると、文S1; ... 文S2;、そして、文F1; ... 文F2;が実行されます。
  5. 捕獲した例外にマッチする catch節が存在しない場合にfinally節が実行されます。
    finally節は記述しなくてもかまいません。finally節を記述すると、catch節で捕獲された例外の場合も含めて、必ずそれが実行されます。
