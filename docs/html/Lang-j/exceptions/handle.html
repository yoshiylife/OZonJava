OZシステムにおける例外の扱い



厳密な例外

OZシステムにおける例外は厳密（precise）です。例外が発生した位置までに存在する文はすべて実行され、式は評価されます。そして、例外が発生した位置より後ろの文は実行されず、式は評価されません。（for文、while文、do文では、ソースプログラムの位置を意味しているわけではありません。実行される順序という意味で、例外が発生した位置よりも後ろの文の実行、式の評価が行われません。）


例外の伝播


try文の複文（上のコードの文1から文n）でメソッド起動を行い、その中でtry文が記述される可能性があります。

発生した例外にもっとも近いtry文のcatch節あるいはfinally節が実行されます。もっとも近いtry文で処理されない場合は、その1つ外側のtry文、というように、捕獲されるまで例外は伝播されます。その例外を捕獲するcatch節がどこにも記述されない場合、実行時のエラーメッセージが出力され、実行が停止します。


例外の伝播 
  cell  Emeralds {
      public void go () {
          dorothy(); 
      }
      void dorothy() {
          scarecrow();
      }
      void scarecrow{
          int i;
          int j = 0;
          i = i / j
      }

  1. メソッドscarecrow で整数をゼロで割っています。システム例外「ArithmeticException」が発生します。
  2. メソッドscarecrow では捕獲されないので、メソッドdorothy に例外が伝播されます。
  3. メソッドdorothy でもこの例外は捕獲されないので、メソッドgo に例外が伝播されます。
  4. メソッドgo もArithmeticExceptionクラスの例外を捕獲しないので、エラーメッセージが表示され、実行が停止します。
