代入演算子（Assignment Operators)



ここでは、OZ言語の代入演算子について説明しています。


OZ言語には、以下の9つの代入演算子があります。

= += -= *= /= %= &&= ||= =|


代入演算子は右結合的（right-associative）です。すなわち、i=j=k は i=(j=k) という意味になります。


代入演算子を用いる式は副作用を持つので、式文に用いることができます。



代入演算子

= 第2オペランドの値を第1オペランドに代入します。
+= 両オペランドの和を計算し左のオペランドに代入します。
-= 両オペランドの差を計算し左のオペランドに代入します。
*= 両オペランドの積を計算し左のオペランドに代入します。
/= 両オペランドの商を計算し左のオペランドに代入します。
%= 両オペランドの剰余を計算し左のオペランドに代入します。
&&= 両オペランドの論理積を計算し左のオペランドに代入します。
||= 両オペランドの論理和を計算し左のオペランドに代入します。
=| 右のオペランドの値を左のオペランドに強制代入します。



LALR(1)文法 
  assign_exp : name assign_operator exp
                    
  assign_operator : =
                    +=
                    -=
                    *=
                    /=
                    %=
                    &&=
                    ||=
                    =|


説明

第1オペランドは、識別子あるいは アトリビュート参照の一次式でなければいけません。

代入演算式の型は、第1オペランドの型です。


強制代入演算子以外の代入演算子は、第2オペランドの型が第1オペランドの型と代入等価でなければいけません。
強制代入演算子は、第2オペランドの型が第1オペランドの型と代入等価であるか、あるいは、第1オペランドの型が第2オペランドの型と代入等価でなければいけません。



単純な代入演算子 =
オペランド 基本型、あるいは、クラス型。

オペランドが基本型の場合には、値のコピーが行われます。
オペランドがクラス型の場合には、リファレンスのコピーが行われます。

式の型 第1オペランドの型

評価 まず、第2オペランドの評価が行われます。そして、その値が第1オペランドの型に変換され、第1オペランドに代入されます。

第2オペランドの型が第1オペランドの型と代入等価でなければいけません。



強制代入演算子 =|
オペランド 基本型、あるいは、クラス型。

オペランドが基本型の場合には、値のコピーが行われます。
オペランドがクラス型の場合には、リファレンスのコピーが行われます。

式の型 第1オペランドの型

評価 まず第2オペランドの評価が行われます。そして、その値が第1オペランドの型に変換され、第1オペランドに代入されます。

第2オペランドの型が第1オペランドの型と代入等価であるか、あるいは、第1オペランドの型が第2オペランドの型と代入等価の場合に評価が行われます。



オペランドが数値型の複合代入演算子： *=, /=, %=, +=, -=
オペランド 数値型。

E1 op= E2 の値は、(E1) op (E2) を評価し、その後、その値をE1の型に型変換されたものに等しくなります。ここで、op は、*、 /、 %、 +、- のいずれかです。

式の型 第1オペランドの型

評価 まず第2オペランドの評価が行われます。そして、op の評価が行われます。その値が第1オペランドの型に変換され、第1オペランドに代入されます。

第2オペランドの型が第1オペランドの型と代入等価でなければいけません。



オペランドが数値型の複合代入演算子： &&=, ||=
オペランド 論理型。

E1 op= E2 の値は、(E1) op (E2) を評価したものに等しくなります。ここで、op は、&&、 || のいずれかです。

式の型 第1オペランドの型

評価 まず第2オペランドの評価が行われます。そして、op の評価が行われます。その値が第1オペランドに代入されます。

（第1オペランドと第2オペランドはともに論理型なので代入等価の問題は発生しません。）





代入演算子の使用例

例: 代入演算子 = と =| 
1:  Array args = sys->getArguments();
2:  if (args != null) {
3:      String buf;
4:      buf =| args->at(0);
5:      if (buf != null) {
6:        port = buf->parseInt();
7:      }
8:  }
  • 1行目で単純な代入が行われます。その代入演算子が評価される前に、メソッド起動式（一次式）が評価されます（sys->getArguments()）。
  • 4行目で強制代入が行われます。その強制代入演算子が評価される前に、メソッド起動式（一次式）が評価されます（args->at (0)）。
  • 6行目でな代入が行われます。その代入演算子が評価される前に、メソッド起動式（一次式）が評価されます（buf->parseInt()）。



例: 代入演算子 =| と += の例 
1:  public locked int update(FMObject obj) {               
2:      Organization o; o =| obj;
3:      int counter = updateWithoutLock(o);             
4:      counter += members->update(o->getMembersPool());
5:      return counter;
6:  }
  • 2行目で強制代入が行われます。
  • 3行目で単純な代入が行われます。
  • 4行目では、メソッド起動式の返り値が変数counterの値に加算されます。
