論理演算子（Logical Operators)



ここでは、OZ言語の論理演算子について説明しています。


OZ言語には、2つの論理演算子があります。

&&
||


&& を論理積演算子（Logical-And Operator）、|| を論理和演算子(Logical-Or Operator)といいます。

論理演算子は左結合的（left-associative）です。すなわち、a && b && c は( a && b) && c と解釈されます。

論理演算子を用いる式は二項演算式を形成します。


論理演算子を用いる式は副作用を持つので、式文に用いることができます。



論理積演算子（Logical-And Operator）

&& 論理積を計算します


LALR(1)文法 
  binary_exp_4 : binary_exp_4 && binary_exp_3
                 binary_exp_3
説明

論理積演算子の2つのオペランドはともに論理型でなくてはいけません。

論理積演算子を用いる二項演算子の型は論理型です。


まず第1オペランドが評価されます。
第1オペランドの値がfalseであれば、第2オペランドの式は評価されません。その場合には、論理積演算式の値はfalseになります。
第1オペランドがtrueであれば、次に第2オペランドが評価されます。そして、最後に論理積演算式の評価が行われます。第2オペランドの値がtrueの場合、論理積演算式の値はtrueです。



例: 論理積演算子 
1:  public Root getValue (String label) {
2:      if (isGreen () && input_data->isRed ()) 
3:          turnGreen (input_data);
4:      return input_data->get (label);
5:  }
  • 2行目に論理積演算子が記述されています。
  • まず、isGreen ()が評価されます。その値がfalseであれば、この式の値はfalseになります。
  • isGreen ()を評価tした値がrueならば、メソッド起動式 input_data->isRed () が評価されます。
    input_data->isRed () の値がtrueならば、この論理和演算式の値はtrueです。
    input_data->isRed () の値がfalseならば、この論理和演算式の値はfalseです。




論理和演算子（Logical-Or Operator）

|| 論理和を計算します


LALR(1)文法 
  binary_exp_5 : binary_exp_5 || binary_exp_4
                 binary_exp_4
説明

論理和演算子の2つのオペランドはともに論理型でなくてはいけません。

論理和演算子を用いる二項演算子の型は論理型です。


まず第1オペランドが評価されます。
第1オペランドの値がtrueであれば、第2オペランドの式は評価されません。その場合には、論理和演算式の値はtrueになります。
第1オペランドがfalseであれば、次に第2オペランドが評価されます。そして、最後に論理和演算式の評価が行われます。第2オペランドの値がtrueの場合、論理和演算式の値はtrueです。



例: 論理和演算子 
  if (lockMode == 0 || (i == 1 && lockMode == 1))
      lockMode=i;
まず、lockMode == 0 が評価されます。
lockMode == 0を評価した値がtrueならば、(i == 1 && lockMode == 1) は評価されません。論理和演算式の値はtrueです。
lockMode == 0を評価した値がfalseならば、(i == 1 && lockMode == 1)が評価されます。その場合には、まず i == 1が評価され、この値がfalseであれば、論理和演算式の値はfalseです。 i == 1を評価した値がtrueであると、lockMode == 1 が評価されます。 lockMode == 1の値がtrueのとき、(i == 1 && lockMode == 1) の値は true となり、論理和演算式の値が true になります。
