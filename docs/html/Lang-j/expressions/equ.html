等値演算子（Equality Operators)



ここでは、OZ言語の等値演算子について説明しています。


OZ言語には、以下の2つの等値演算子があります。

== !=


等値演算子は左結合的（left-associative）です。すなわち、i==j==k は( i==j)==k)と解釈されます。


等値演算子を用いる式は二項演算式を形成し、式の値は論理型です。




等値演算子

== 第1オペランドの値と第2オペランドの値が等しいとき true、それ以外は false の値を持ちます。
!= 第1オペランドの値と第2オペランドの値が等しくないとき true、それ以外は false の値を持ちます。



LALR(1)文法 
  binary_exp_3 : binary_exp_3 binary_operator_3 binary_exp_2
                 binary_exp_2
                    
  binary_operator_3 : ==
                      !=


説明

等値演算子の2つのオペランドの型は同一で、数値型、文字型、論理型、クラス型のいずれかでなければいけません。

式の値は、論理型になります。


オペランドの型が数値型の場合はオペランドに対して型の昇格が行われます。
float型、あるいは、double型であれば、浮動少数等値比較（a floating-point equality test）が行われます。その他の数値型であれば整数等値比較（an integer equality test）が行われます。（IEEE 754 準拠）

オペランドの型が論理型の場合の == の評価は、両オペランドがともにtrue、あるいは、ともに falseの場合に true になります。
オペランドの型が論理型の場合の != の評価は、片方が true で他方が false の場合に true になります。

オペランドの型がクラス型の場合は、もう一方のオぺランドとして null リテラルだけが可能です。
クラス型のオペランドの値がnull のときに、式の値は true になります。




例: 等値演算子 オペランドが基本型の場合 
  char fileSepChar;  
      ...
  if (fileSepChar == '/') {
      cmd => breed("oz -f ");
  } 
      ...
  • char型の変数 fileSepCharの値が文字 / に等しいとき、式の値はtrueになります。



例: 等値演算子 オペランドがクラス型の場合 
1:  void refreshServerGOL(String gol)  {
2:      if (gol != null) {
3:          serverGOL = gol;
4:          serverGOLView->setText(serverGOL);
5:      }
6:  }
  • 2行目で等値演算子が使用されています。オペランドの型がクラス型の場合は、もう一方が null リテラルだけが可能です。gol が null でなければ式の値はtrueになります。
