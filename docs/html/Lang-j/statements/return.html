return 文（The return Statement）


return文によってメソッド起動を制御できます。

return [式] ;

式を省略することができます。式が省略できるのは、メソッド宣言の返り値が void と宣言されている場合です。


LALR(1)文法 

  jump_stmt : RETURN exp_or_empty ;

  exp_or_empty : /* empty */
                 exp



説明
return文はメソッド本体の定義、あるいは、コンストラクタ本体の定義の中で使用されます。

式が省略された場合には、return文によって返される値はありません。したがって、メソッド宣言の返り値が void と宣言されていなくてはいけません。

式を持つreturn文では、メソッド宣言でその式の値の型が返り値として宣言されている必要があります。この場合、return文の式でその型の値や変数が指定されていなくてはいけません。


まず、式の評価が行われ、それが正常に評価されると、その値を返り値として、起動されていたメソッドから呼出しを行ったメソッドに処理を移します。



return 文に記述された式について、その式とその式が記述れたメソッドを呼び出したメソッドの返り値型は代入等価でなくてはいけません。

メソッドの返り値がない場合 (void)
式を持たない return 文のみが有効です。

コンストラクタ
式を持たない return 文のみが有効です。

その他の場合
return 文に式が必要です。
その式の型とメソッドの返り値の型 は代入等価でなければいけません。


返り値を持つメソッドの実装の最後に実行されるものとして記述 した文は return 文でなければいけません。


例: 式を持つreturn文が必要なメソッド本体の定義 

  public DeliveryFile leaveToSearchAndDelivery(String codePath){
      DeliveryFile df = searchAndDelivery(codePath); /* first myself*/
    
      if( df != null ) return df;
      int RetryNum = 0;
      String targetCDAGOL;
    
      while( (targetCDAGOL = targets->next(RetryNum)) != null ){
          ClassDeliveryAgent targetCDA; targetCDA => bind(targetCDAGOL);
          df = targetCDA->searchAndDelivery(codePath); /* next others */
          if(df!=null) break;
          RetryNum++;
      }
      return df;
  }
  • return文の式 df の型は、DeliveryFile です。



例: 式を持つreturn文が必要なメソッド本体の定義 

  public Iterator getLabels () {
    return input_data->keys ()->iterator ();
  }



例: return文がなくてもよいメソッド本体の定義 

  public void quiet()  {
      core->quit();
      flushCell();
  }
  • 返り値がvoidなので、return文がなくてもかまいません。
