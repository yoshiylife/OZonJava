try 文（The try Statement）


状況によって文の実行を制御できます。

try 複文1 [catch(型名 変数名)｛文｝［,..., finally 複文］]


LALR(1)文法 

  try_stmt : TRY compound_stmt try_body
                    
  try_body : catch_list
             catch_list FINALLY compound_stmt
                    
  catch_list : /* empty */ 
               catch_list CATCH ( type_name IDENTIFIER ) { stmt_list }



説明

try節に記述された複文には、このtry文で処理する文を記述します。

catch節では例外処理を記述します。catch 節に宣言する変数名は、発生した例外を表わし、その catch 節内で参照するために用いられます。この変数名のスコープはそのcatch 節内です。

finally 節は try 節の実行後必ず最後に実行されます。つまり、

  • try 節の実行により例外が発生しない場合は、try 節の実行に引き続 き finally 節が実行されます。try 節の実行途中に 移動文（break文、continue文）が実行され制御が try 節の外に移動する場合にも finally 節は実行されます。
  • try 節の実行により例外が発生した場合、

    その例外を受け取る catch 節が存在しない場合は、finally 節が 実行し、現在のメソッドの呼び出しもとに例外が伝播します。

    その例外を受け取る catch 節が存在する場合は、その catch 節が 実行された後、finally 節が実行されます。catch 節の実行途中に 移動文 の実行により、制御が catch 節の外に移動する場合 にも finally 節は実行されます。



例: try文 

 1: try {
 2:     ZipArchive zip=>create (source);
 3:     Iterator it = zip->list ()->iterator ();
 4:     while (it->hasMoreElements ()) {
 5:         String ozsrc;
 6:         ozsrc =| it->nextElement ();
 7:      
 8:         sys->debugPrintln (ozsrc);
 9:       
10:         File f=>breed (ozsrc);
11:         String src = createPath (f->getName ());
12:      
13:         zip->extract (src, ozsrc);
14:         sources->add (src);
15:     }
16: } catch (ZipException e) {
17:     sys->debugPrintln (e->getMessage ());
18:
19:     throw source;
20: }
  • 2行目から14行目までが実行されます。その実行中で例外が発生するとcatch節で処理されます。
  • この場合には、ZipException型のオブジェクトを例外として捕獲して処理する記述が指定されています（16行目）。
  • 発生した例外の型がZipExceptionであると、17行目から19行目が実行されます。16行目で、例外は変数eで表現されています。17行目で、変数eで表現される例外オブジェクトにメッセージを送っています。
