型の等価性と代入変換規則



代入等価

代入演算子（=）における操作を代入と呼びます。そして、型 A の要素 a を型 B の要素 b に代入可能である場合、A は B に代入等価であるといいます。

OZ言語では、代入等価を以下のように規定しています。

  1. 同一型は代入等価です。
  2. 基本型とクラス型は代入等価ではありません。
  3. 数値型、文字型、論理型、condition型は互いに代入等価ではありません。
  4. 整数型は浮動小数型に代入等価です。
  5. byte 型は、short 型、int 型、long 型に代入等価です。
  6. short 型 は int 型、long 型に代入等価です。
  7. int 型は long 型に代入等価です。
  8. float 型は double 型に代入等価です。
  9. クラス型T の子孫クラスの型S は T に代入等価です。


上記の1.は「恒等変換」です。4.ー8. は「基本型のワイドニング変換」です。9.は「クラス型のワイドニング変換」です。



代入変換規則

次に、代入に関するOZ言語の規定について説明します。
説明のため、型 A の要素 a の型 B の要素 b への代入を例としてとりあげます。以下の説明の強制代入とは、強制代入演算子（|=）を使用する代入を意味しています。強制代入は、クラス型に対して有効な操作で、実行時に代入等価が評価されます。

  • A と B が同一型であれば、代入可能です。

  • 代入が強制代入でない場合、A が B に代入等価でなければ代入を行うことができません。

  • 代入が強制代入の場合、A が B に代入等価であるか 、あるいは、B が A に代入等価でなければ、代入を行うことができません。




代入に関する特別な規則

整数定数が代入演算の右オペランドの値の場合には、上記の規則に加えて以下の条件をすべて満たせば代入等価であり、代入を行うことができます。

  • 左オペランドの型が byte 型または short 型 または int 型である。
  • その整数定数の値が、左オペランドの型で表現可能である。




switch 文の判定式および case の式

  • switch文の式は、その型が int 型に代入等価な型の式でなければいけません。
  • caseラベルの式は、すべてのオペランドが整数定数であるような式でなければいけません。
