アトリビュート宣言


ここでは、アトリビュート宣言、アトリビュート名のスコープ、アトリビュートの修飾子について説明します。


OZクラスのアトリビュートは次の宣言文で宣言します。

[アクセスレベル] [修飾子] 型名 アトリビュート名;


アクセスレベルと修飾子は省略することができます。その場合には、省略時の指定が行われます。



アクセスレベル

以下の指定ができます。

protected 定義クラスのオブジェクトのメンバおよび子孫クラスのオブジェクトのメンバからアクセスできます。
（なし） 定義クラスのオブジェクトのメンバからアクセスできます。



修飾子

以下の指定ができます。

final コンストラクタ本体の定義文でのみ、このアトリビュートへの代入を行うことができます。
transient オブジェクトが永続化される場合にこのアトリビュートは切り離されます。
オブジェクトが再起動された後の値は保証されません。



型名

アトリビュートの型を指定します。
アトリビュートの型は基本型あるいはクラス型です。


アトリビュート名

アトリビュート名は同一クラス内で重複してはいけません。すなわち、アトリビュート名のスコープはその宣言文のクラスです。




例: アトリビュート宣言 

 1: class NameDirectory {
 2:     protected Dictionary SubDirs;
 3:     protected Dictionary Entries;
 4:     protected bool branch;
        ．．．
 n: }
  • 2行目にDictionary型（クラス型）のアトリビュートが宣言されています。アトリビュート名は SubDirs です。
  • 3行目にDictionary型（クラス型）のアトリビュートが宣言されています。アトリビュート名は Entries です。
  • 4行目には論理型のアトリビュート branch が宣言されています。
  • この3つのアトリビュートはアクセスレベルが protected なので、同一クラスあるいは子孫クラスのオブジェクトからアクセスできます。




例: アトリビュート宣言 

 1: class FMObject : *Root {
 2:     protected Pointer identifier;
 3:     String name;
 4:     Set relations;
 5:     RelationComparator rc;
        ．．．
 n: }
  • 2行目に宣言されているPointer型（クラス型）のアトリビュート identifier は同一クラスあるいは子孫クラスのオブジェクトからアクセスできます。
  • 3行目にString型（クラス型）のアトリビュート name が宣言されています。これは継承されないアトリビュートなので、同一クラスのオブジェクトからのみアクセスできます。
  • 4行目にはSet型（クラス型）のアトリビュート relations が宣言されています。同一クラスのオブジェクトからのみアクセスできます。
  • 5行目にはRelationComparator型（クラス型）のアトリビュート rc が宣言されています。同一クラスのオブジェクトからのみアクセスできます。



LALR(1)文法 


attribute_declaration : 
    access_modifier  modifier_list  type_name  identifier_list ;


access_modifier :
     /* empty */
     PUBLIC
     PROTECTED


modifier_list : 
    /* empty */
    modifier_list modifier


modifier : 
    FINAL
    STATIC
    LOCKED
    ONCE
    ABSTRACT
    TRANSIENT


type_name :
    BYTE
    CHAR
    SHORT
    INT
    LONG
    FLOAT
    DOUBLE
    BOOL
    CONDITION
    IDENTIFIER
    CLASS IDENTIFIER
    CELL IDENTIFIER

identifier_list : 
    identifier_list , IDENTIFIER 
    IDENTIFIER
