クラスの継承


ここでは、ＯＺクラスの継承に関する概念について説明します。



上位クラス

クラスの宣言は、

[修飾子] class 識別子［上位クラス］ { メンバリスト};

と記述しますが、インタフェース部、動作部を継承する既定義のクラスを上位クラスと呼びます。
ここでインタフェース部とは、

  • そのクラスに宣言されている publicメソッドのシグネチャと戻り値の型
  • そのクラスの上位クラス（親クラス）

であり、動作部はそれ以外の部分、すなわち、

  • アトリビュート
  • メソッド本体の定義
  • public以外のアクセスレベルを持つメソッドのシグネチャと本体定義
  • コンストラクタのシグネチャと本体定義

です。

OZ言語では、0 個以上のクラスを上位クラスと して宣言することができます。上位クラスの宣言が行われない場合には暗黙のクラスを上位クラスとみなすことになります。（Rootクラスがこの暗黙のクラスです。）
上位クラスには上位実装クラスが含まれます。上位実装クラスの動作部を除いて、上位クラスはクラス宣言対象のクラスのインタフェース部になります。



上位実装クラス

インタフェース部と動作部の両方を継承する上位クラスを 上位実装クラス と呼びます。上位実装クラスと して宣言できるクラスはたかだか1つです。

クラスのインタフェース部の宣言には、上位クラスの宣言が含まれています。上位実装クラスの動作部は、そのクラスのインタフェース部の宣言には含まれません。上位クラスの動作部はそのクラスの動作部に含まれることになります。



祖先クラス、子孫クラス、下位クラス

継承と関連して、祖先クラス、子孫クラス、下位クラスというOZの概念があります。

上位クラスが各々のクラス宣言において上位クラスを宣言することができるので、上位クラスの再帰的連鎖が形成されます。上位クラスの連鎖に含まれるクラスを総称して、祖先クラス と呼びます。逆に、上位クラスを宣言したクラスはその上位クラスに対する 下位クラス と呼びます。すべての下位クラスの連鎖に含まれるクラスを 総称して、子孫クラス と呼びます。

子孫クラスのいずれかを祖先クラスとするような継承関係を宣言することはできません。したがって、OZクラスは1つのクラス、すなわち、Rootクラスをルートとする階層構造を形成します。



クラスの継承

下位クラスは上位クラスからインタフェース部を継承します。また、上位実装クラスから priveteメンバ以外を継承します。継承したメソッド（シグネチャ、あるいはシグネチャと本体）を再定義することができます。
再定義とは、継承したメソッドのメソッド本体の定義を新たに行うことです。上位クラスが定義しているメソッドのシグネチャと同一のシグネチャを持たなくてはいけません。
