  • OZ 認証サーバの設定/起動方法
OZ 認証システムを利用するための設定
machine.cipher の設定
次のエントリを作成する。
JP.go.ipa.oz.crypt.DES.OzDESCipher
GOD/.cipher の設定
次のエントリを作成する。
JP.go.ipa.oz.crypt.DES.OzDESCipher
パスワードファイルの設定
次のファイルに、パスワードを書く。
$OZHOME/secure/password
パスワードは、任意の (タイプできる) 文字列でよいが、 空白や改行を含んではならない。
パスワードファイルは、他のユーザからは秘匿せねばならない。 マルチユーザの環境では、ファイルやディレクトリのパーミッションを 操作して、秘匿するようにする。
OZ 認証サーバの設定/起動方法
認証サーバ起動アドレスの設定
認証サーバの管理下の全ての OZ ホームの $OZHOME/lib/oz.propertiesに以下を追加する。
oz.authentication.server=認証サーバのIPアドレス:ポート番号
認証サーバの起動
oz as
で認証サーバを起動する。 認証サーバを起動するのは特権ユーザ（管理者）のみである。
認証サーバへのユーザの登録
特権ユーザは認証対称となるユーザから 郵便書留等安全な方法で秘密鍵を受け取る。 特権ユーザによる認証サーバへの操作は認証サーバと同じOZHOMEでなければならない。
次に以下のコマンドで認証サーバを起動し、ユーザ名、秘密鍵を登録する。
addUser userName userKey
このとき与える userKeyは OZHOME/secure/passwordの内容と同一でなければならない。
次に以下のコマンドでグループを登録する。
addGroup groupName ownerName member1 ... memberN
この他メンテナンスのために以下のコマンドが用意されている。 （引数の与え方は -h で参照のこと）
ユーザ管理用
removeUser：ユーザを削除
changeKey ：ユーザの秘密鍵を変更 showUser ：登録ユーザ一覧を表示
グループ管理用
removeGroup：グループを削除
changeOwner：グループのオーナを変更
addGroupMember：グループにメンバを登録
removeGroupMember：グループのメンバを削除
showGroup ：登録グループ一覧を表示
