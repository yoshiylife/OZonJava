CDAの最も単純な使い方

  1. OZを起動させる前、libディレクトリのoz.propertiesに二つのプロパティ を追加する。

    まず、プロパティoz.classloader.deliveryAgent.urlは、executorとCDAの間のソケット通信に使うソケット(ホスト名とポート番号)を指定するものである。 例えば、executorを動かすマシンのホスト名がhost1で、このhost1のポート番号6010 のソケットを指定するのなら、次のようなプロパティ設定になる。
    oz.classloader.deliveryAgent.url=ozclass://host1:6010/
    ここで、ozclassとはそのソケット通信のプロトコルの名である。

    次に、プロパティoz.cda.targetである。 executorは実行に必要なクラスがローカルリポジトリに見当たらない場合は、 ローカルシステム上のCDAに対してそのクラスをリモートシステムから検索して くるように要求する。 CDAは検索方針に従っていくつかのリモートシステムのCDAに順番に打診を行なっていく。 ここで検索方針とは、打診を行なっていく順番 であり、打診とは、リモートシステム のリポジトリに目的のクラスがあるかどうかを聞くことである。 プロパティoz.cda.targetの値はこの検索方針である。 すなわち、打診を行なっていくリモートシステムのドメイン名:OZHOME識別子を','をデリミタとして並べたものである。 例えば、oz.ipa.go.jp:remoteoz、oz.ipa.go.jp:farozという順で打診を 行なっていくのなら次のようなプロパティ設定になる。
    oz.cda.target=oz.ipa.go.jp:remoteoz,oz.ipa.go.jp:faroz
    尚、このプロパティoz.cda.targetを設定しない場合は、クラスの検索は行なわれない。
  2. CDAを動かすにはリゾルバが必要である。リゾルバの設定を リゾルバのドキュメントに従って適切に行なっておく。 設定の際には、ポート番号を上述のozclassプロトコル によるソケットのポート番号にぶつからないように注意。
  3. CDAは"cda"というGODのセルであるので、CDAの起動・停止はセルcdaの起動・停止である。
