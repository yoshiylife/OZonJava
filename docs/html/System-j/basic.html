OZシステムの基礎知識

ここではOZシステムを利用するための基礎知識を紹介する。

OZHOME

INSTALL[.JP]に従ってインストールするとひとつの ディレクトリができあがる。 システムを実行する前にこのディレクトリを環境変数OZHOMEに登録する。 このディレクトリをOZHOMEと呼ぶ。

OZHOMEはひとりがひとつづつ所有すべきものである。 利用者がふたりいればOZHOMEはふたつ作らなければならない。

OZ-VM

OZはバイトコードマシン上でプログラムを実行するシステムである。 このバイトコードマシンをOZ-VM(Virtual Machine)と呼ぶ。 OZを実行するにはOZ-VMを実行する。 OZ-VMはozという名前のプログラムである。 
        例:
        % oz ide
最初の引数はセルの名前である。 ここで指定できるのは$OZHOME/objects以下にあるディレクトリ名である。 ideは統合開発環境である。 launcherはプログラムを実行するランチャーである。 通常このどちらかを指定すれば用は足りるはずである。

セル

OZはオブジェクトを実行するシステムである。 OZのオブジェクトはふたつの種類がある。 ひとつはプログラム中に現れるオブジェクト指向で言うところの いわゆるオブジェクトである。 このオブジェクトはプログラムの内部にしか現れないので、ここでは述べない。 もうひとつはオブジェクトの集合をオブジェクトとして実現したもので セル、あるいはグローバルオブジェクトと呼ばれる。 (名前が2種類あるのは開発の歴史的経緯による)

セルは永続オブジェクトである。 すなわちファイルとして保管することができる。 この永続ファイルが$OZHOME/objects以下にあるディレクトリである。 OZ-VMは起動時にファイルを読み込みセルとして稼働する。

GODとGOL

$OZHOME/objects以下にセルのイメージファイルが置かれる。 $OZHOME/objects相対のパス名をGOD(Global Object Directory)と呼ぶ。 GODはOZHOME内におけるセルのIDである。

GODはセルの通信アドレスの一部となる(参照)。 OZシステムではOZHOMEもアドレスを持つ。 これをGOL(Global Object Location)と呼ぶ。 GODはGOLの一部である。
