プロジェクトマネージャとスクールブラウザ


OZ IDE を起動すると、プロジェクトマネージャとスクールブラウザが表示されます。2つのウィンドウに別れていますが、この2つのウィンドウは対になって動作します。

プロジェクトマネージャ

プロジェクトとはアプリケーション構成するさまざまな要素のことです。OZではクラスライブラリやアプリケーションをサブジェクトという単位で構築します。プロジェクトマネージャを利用すると、サブジェクトを構成するクラスのソースファイルの編集やコンパイルを行うことができます。また、サブジェクトのリリースパッケージを作成することもできます。

プロジェクトマネージャは複数のプロジェクトを管理することができます。同時には１つのプロジェクトだけを開いて、そこでそのプロジェクトの編集、コンパイルなどの作業を行うことができます。

現在は1つのプロジェクトは1つのディレクトリと対応づけるようになっています。プロジェクトマネージャを利用するサブジェクト開発では、まず、プロジェクトを作成し、そのプロジェクトと対応付けられるディレクトリを設定してください。そして、そのディレクトリにソースファイルを格納、編集してください。

ソースファイルのコンパイルのためにはスクールが必要です。スクールブラウザに新規サブジェクトを登録して開発したクラスを登録、スクール情報を作成してください。

この設定後で自動コンパイルやリリース用コンパイルを含めてさまざまなコンパイルを行うことができます。


スクールブラウザ

スクールとはクラス名とクラス識別子を対応づける役割を持つものです。OZでは、これによってクラス名の名前空間の問題を解決しています。そして、OZコンパイラはスクール情報によってクラス間の相互参照を解決しています。

スクールブラウザによって個々のクラスのスクール情報を設定し閲覧することができます。さらに、クラスブラウザとサブジェクトカタログブラウザを開くことができます。


プロジェクトマネージャとスクールブラウザの連動

スクールブラウザはスクール情報を設定、閲覧するだけではなくプロジェクトマネージャと連動するので注意が必要です。コンパイルをするときには、プロジェクトと対応するサブジェクトが閲覧されていなくてはいけません。



アプリケーションやツールを作成する一連の作業は「OZ IDEを利用するアプリケーション開発」を参照してください。
