自動コンパイル


サブジェクトを構成するクラスの修正を判定して修正クラスを含むファイルだけを再コンパイルします。


<自動コンパイルを行うための事前条件>
自動コンパイルを行う前に以下の作業が行われている必要があります。
  1. 対象サブジェクトの全クラスを格納しているソースファイルがプロジェクトディレクトリに格納されている。
  2. そのソースファイルに含まれる全クラスをサブジェクトに登録し、インタフェース識別子と動作部識別子を付与している。
  3. ソースファイルのコンパイルのために必要なサブジェクトを選択している。
  4. スクールブラウザのカレントサブジェクトがコンパイル対象サブジェクトになっている。

1つのプロジェクトに対してさまざまなコンパイルを行うことができますが、3．のサブジェクト選択処理は1度行えば毎回行う必要はありません。したがって自動コンパイルが選択されるときには、この処理は行われているでしょう。サブジェクト選択については、「クラスコンパイル」、「インタフェースコンパイル」、「サブジェクトコンパイル」、「リリース用コンパイル」を参照してください。


<OZソースファイルの自動コンパイル>
１．プロジェクトマネージャの Compile メニューから compile auto メニュー項目を選択します。

コンパイル情報を表示するウィンドウが表示されます。

自動コンパイルでは、コンパイルが成功したかどうかのメッセージだけが返されます。
MyDialog.oz compiled successfully."
Quit.oz compiled successfully."
Test.oz compiled successfully."		 (すべて成功した場合の例)


コンパイルが終了すると、OKボタンがアクティブになります。 このボタンをクリックするまで、コンパイル情報を表示するウィンドウは表示されています。
