以下にファシリティ管理が提供する基本的なサービスの概要を示す。

  • ファシリティ管理 API
    ファシリティ管理における基本的な管理対象であるリソースを表 象するクラスであるFMObjectを操作するために、利用者、および テンプレートを経由して利用者により定義される個別の FMObject(の子孫)の両者に対し、統括的に提供される ファシリティ管理API(Application Program Interface)である。
    FMObjectを取り扱う為の基本的操作として、
    • FMObjectの生成
    • FMObjectの削除
    • FMObjectの管理区域間の移動
    • FMObject間の関係付け
    等のサービスが提供される。

  • メンテナンス機能
    ファシリティ管理において、リソースはそれぞれ管理主体に属し、そこ で管理される。管理主体において管理対象であるリソースを矛盾無く管 理するためのサービスを提供するのがメンテナンス機能である。

  • リソースの予約、予約の照会/キャンセル機能
    リソースの予約、照会、そして予約のキャンセルを行うため の基本サービスのことである。時間軸上の関連に特 化した予約、照会、および予約のキャンセルが可能となっている。

  • スケジューリング機能
    前項のリソースの予約に関する機能を元に、リソース自 身を時間軸上の「予約可能なモノ」として取り扱うことにより、その スケジューリングを行うサービスが提供される。

  • リソースの時間的管理
    ファシリティ管理では、その管理対象であるリソースに対し、発生した 事象(例えば生成、他のリソースとの関連の結び付き、など)について、 時間的順序に従った記録が行われる。リソースに対応したこれら記録を 参照することができる。

  • リソースの組織管理
    他のリソースを組織的に管理することを可能とするために、組 織を表象するリソースが提供されるている。組織リソースにより リソース相互の包含関係を利用者が自由に構成することが可能となる。
