機能詳細

リソースの予約、照会、そして予約のキャンセルに就いて述べる。以下 ではこの予約、照会、および予約のキャンセルを指して「予約一般」と呼ぶ。
予約一般は、その表現をリソース相互の関連に帰着することができるが、 ここではファシリティ管理機構の提供するフレームワークを利用して、時間軸上の関連に特 化した予約一般の実現を示す。
予約一般もやはりリソース間の関連であり、従ってファシリティ管理機構の提供する リレーションによる関連の実現機能を直接用いることで表現することも可能 である。しかし予約一般の場合にはその関連の生成される頻度、またその関連 自体の存在の揮発性いずれもが高いと云う点で、通常のリレーションによる 表現とは異なる特化された実現形式が効果的と云える。予約一般を取り扱うた めに以下の仲介のための機構を提供する。

予約可能な実体の総称表現
 
ある一つの予約一般において、当事者として出現するエンティティを以下の2 者に分類して取り扱う。但しエンティティに対するこの分類は飽くまでもその 予約一般を限りに一過的に取り扱われる必要がある。当事者エンティティへの 不可逆的なリレーションの構築を意味するものではない。
  • 予約される側の当事者を代理する「予約可能なモノ」
    予約される側の当事者であるエンティティを総称すると同時に、実際 に「予約可能なモノ」として振る舞う。これはさらに以下の2つのことを行 う。
    1. 予約対象としての「基質」、およびその予約「基底単位」の定義
      ここでは時間軸上での予約一般に対象を絞っているので予約は線形であり、各々以下のようになる。
      • 基質： 時間(固定)
      • 単位： 具体化時に指定(省略時 の標準を設けても良い、例えば30分単位)
    2. 当該予約基質への操作
      具体的に当該基質上での線形な予約を表現する上での操作を提供する
      • 連続した基質の取得
      • 取得されている基質の認識
      • 取得した基質の解放
  • 予約する側の当事者を代理する「予約」
    予約する側を総称すると同時に、予約の具体的な要求内容を表現する。 従ってこの両代理者はそれぞれお互いに意味的に依存していることになる。
    1. 予約条件、ここでは連続時間範囲
    2. 予約成立時の状態の維持
    3. 予約解消要求への対処

予約一般の管理
 
上記の「予約可能なモノ」を集合として管理し、それらリソースへの 「予約する側の当事者」からの要求を受け付け、実際の予約要求を仲介する。 また「予約の照会」はここで提供されるサービスである。
