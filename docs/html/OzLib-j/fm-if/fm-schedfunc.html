機能詳細

スケジューリングの機能に就いて述べる。 スケジューリング自体は、「リソースの予約、予約の照会/キャンセル機能」において言及された リソース予約一般を可能とする機構を利用 した一種のクライアント、ないし特定のアプリケーションである。
従ってリソース予約一般がそうであったように、ここでも スケジューリング機能は時間軸上でのそれをスコープとしている。 スケジューリングを行うにはその準備を以下の順で行う。

  1. スケジュール管理を望むFMオブジェクトは、自分自身に就いて「予約可能 なモノ」として自身を分類する。
  2. 同時に「予約可能なモノ」となった自分自身に対し、「予約管理者」と して働きかける。

この準備ができていれば、実際のスケジューリングの利用は以下のように行われる。

  1. 具体的なスケジュール項目は、その内容をそれぞれ「予約」として生 成し、「予約可能なモノ」である自分自身に対し予約する。
  2. スケジュールの参照は予約管理者として「予約可能なモノ」の予約状 況を照会すればよい。
