機能詳細

リソースの組織管理に就いて述べる。
一般論として組織とは、特定の包含構造に着目し、それを主体として取り扱う ことを可能としたものと捉えることができる。従って以下のことが行えなけれ ばならない。

包含構造を表現できること
 
ファシリティ管理機構において、リソースであるFMオブジェクト(の集合)を管理する一般的、 且つ基本的機構としては既にプールと呼称されるクラスが提供されて いる。これは自分自身もFMオブジェクトとして実現されていることから、この プールを用いることにより、リソースの世界、すなわちFMオブジェクトの インスタンスのレベルにおいて、普遍的な包含構造の実現が可能となっ ている。
従ってリソースの組織管理においても、プールにより提供されるこの 機能を基本的に敷衍することにより、包含構造を表現する。
以下の機能が提供される。
  • 組織への人の配置
    組織には人が配置できなければならない。ここで人とはファシリティ管理 機構において人格権を有するリソースの総称である。従って組織自体もそ の対象となる。
  • 組織からの人の除外
    配置された人は何時かは組織から除外される。
  • 配置されている人の確認
    組織は、そこに誰が配置されているのか確認できなければならない。
  • 配置されている人の呼び出し
    組織内に配置されている人を呼び出すことができなければならない。

主体として振る舞えること
 
組織を受動的な客体として扱うだけであるなら、前項に述べた事柄より、単純 にプールと云うリソースをそのまま直接利用することで同等のことが 行える。ここでは組織としてそれ自身が主体として振る舞うことを可能とする ために、さらに人格権を付与する。
人格権はファシリティ管理機構においてアクタとして抽象表現されている。組織の人格 権はいわゆる「法人」格であり、従ってアクタそのものとしてではなく、 その特殊化された形式で表現されるべきである。また当然、個人の人格権とも 識別できることが必要である。
以下の機能が提供される。
  • 役割の代替
    組織は特定の役割を果たすために組織される。従って組織それ自体は役割の代 替、すなわち特定の役割の換喩として扱える必要がある。ファシリティ管理機構では役割の 総称表現もアクタにより行われており、これを敷衍する。
  • 組織への配置
    組織は人格権を有するリソースであり、従ってそれ自体が人の場合と同 様に自分以外の組織に配置される。

その他
 
当然、組織の生成、消去が可能でなければならない。
