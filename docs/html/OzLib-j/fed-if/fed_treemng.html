フェデレーションの階層的管理

引数に入ってくるフェデレーションの GOL を元に、 まず当 Autonomy セルが管理している Federation 情報の集合を検索し、 登録または削除を行う。
更新は、 Federation セルと Autonomy セルの両方で 再帰的に行い、お互いの保持する情報の整合性を維持する必要がある。 そのために、実装においては引数の GOL をバインドし、 GOL に対応する Federation セルの更新用のインタフェースを 呼び出すこととする。これは再帰的な呼び出しとなるが、 集合の要素を判定することで、無限ループによるデッドロックを 回避することとする。

  • public void Autonomy#attachToFederation(String gol)
    引数として Federation セルの GOL をとる。引数 gol が、 Autonomy 内の Federation 集合に含まれていなければ、 集合に追加する。
    さらに、引数で指定された Federation セルの Autonomy 集合に 当 Autonomy セルの GOL を Federation#addAutonomy(String) により 追加する。この処理は相互に再帰的に行われるので、 デッドロックを避けるために最初に登録状況を判定する必要がある。

  • public void Autonomy#detachFromFederation(String gol)
    引数として Federation セルの GOL をとる。引数 gol が、 Autonomy 内の Federation 集合に含まれていれば、 集合から削除する。
    さらに、引数で指定された Federation セルの Autonomy 集合から 当 Autonomy セルの GOL を Federation#removeAutonomy(String) により 削除する。この処理は相互に再帰的に行われるので、 デッドロックを避けるために最初に登録状況を判定する必要がある。
