OZ タイマライブラリ

タイマをかけて実行したいメソッド呼び出しごとに、ライブラリ内の MethodRunnable というクラスを継承した Runnable クラスを用意する。
本線のスレッドと、この Runnable クラスから生成されるスレッドが連動する ことによりタイマが実現されている。
このタイマの使い方には二通りある。今、タイマをかけて実行したいメソッドが 定義されているクラスをAとする。一般的に、タイマのユーザがクラス A の定義 を変更することはできない(例えば、クラス A を定義した人がタイマユーザと別 の人の場合など)。そのような場合の使い方が「タイマの使い方 1」である。 そして、タイマのユーザがクラス A の定義を変更できる場合の使い方が「タイマ の使い方 2」である。
